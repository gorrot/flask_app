## 项目概述

### 项目名称
Flask检测数据服务器

### 项目简介
这是一个基于Flask的RESTful API服务器，用于接收、存储和提供工业设备检测数据。主要功能包括：
- **检测数据接收**：接收来自PIVdata2.py的检测数据（磨煤机状态、皮带状态、负荷监控等）
- **数据查询**：为Android APP提供检测数据查询接口
- **统计分析**：提供数据统计和皮带状态查询
- **传感器数据**：集成外部传感器平台，提供实时传感器数据

### 技术特点
- **DDD分层架构**：采用领域驱动设计，代码组织清晰
- **内存存储**：使用内存存储，快速响应
- **向后兼容**：API接口保持稳定，无需修改客户端代码
- **错误处理**：完善的错误处理和日志记录

### 应用场景
- 工业设备监控数据中转
- Android移动端数据服务
- 实时传感器数据代理
- 检测数据统计分析

## 架构设计

### DDD分层架构

本项目采用**领域驱动设计（Domain-Driven Design）**的分层架构：

```
Interface Layer (接口层)
    ↓ 依赖
Application Layer (应用层)
    ↓ 依赖
Domain Layer (领域层) ← Infrastructure Layer (基础设施层)
```

### 各层职责

#### 1. Interface层（接口层）
- **位置**：`interface/`
- **职责**：处理HTTP请求/响应，协议转换
- **特点**：薄层设计，不包含业务逻辑
- **文件**：
  - `api.py` - Flask路由定义
  - `schemas.py` - 请求/响应DTO

#### 2. Application层（应用层）
- **位置**：`application/`
- **职责**：实现业务用例，协调领域对象
- **特点**：用例服务，事务边界管理
- **文件**：
  - `detection_service.py` - 检测数据用例
  - `stats_service.py` - 统计用例
  - `belt_service.py` - 皮带状态用例
  - `sensor_service.py` - 传感器数据用例

#### 3. Domain层（领域层）
- **位置**：`domain/`
- **职责**：核心业务逻辑，实体和规则
- **特点**：不依赖其他层，最内层
- **文件**：
  - `entities.py` - 实体（Detection）
  - `value_objects.py` - 值对象（DetectionType, DetectionStatus, Timestamp）
  - `rules.py` - 领域规则和工厂方法

#### 4. Infrastructure层（基础设施层）
- **位置**：`infrastructure/`
- **职责**：技术实现，数据存储和外部服务
- **特点**：可替换的技术实现
- **文件**：
  - `repository.py` - 仓储实现（内存存储）
  - `sensor_client.py` - 外部传感器服务客户端

### 依赖关系原则

1. **领域层不依赖任何层**（最内层）
2. **应用层依赖领域层和基础设施层**
3. **接口层依赖应用层**
4. **基础设施层实现领域层定义的接口**

## 功能模块

### 1. 检测数据模块

**服务类**：`DetectionService`

**功能**：
- 接收检测数据（来自PIVdata2.py）
- 查询检测数据（供Android APP使用）
- 标记数据为已读

**数据来源**：
- PIVdata2.py通过`POST /receive_detection`发送检测数据

**数据流向**：
```
PIVdata2.py → Flask服务器 → Android APP
```

### 2. 统计模块

**服务类**：`StatsService`

**功能**：
- 统计总检测数
- 统计未读检测数
- 按类型统计
- 按任务（URL+User）统计

**使用场景**：
- Android APP显示统计信息
- 数据分析和监控

### 3. 皮带状态模块

**服务类**：`BeltService`

**功能**：
- 解析皮带状态消息
- 提取每个皮带的最新状态
- 提供皮带状态查询接口

**业务规则**：
- 从`belt_status`类型的检测数据中提取状态
- 保留每个皮带的最新状态（按时间戳）

### 4. 传感器数据模块

**服务类**：`SensorService`

**功能**：
- 获取传感器访问令牌
- 获取实时传感器数据
- 数据格式转换（扁平化）

**外部服务**：
- 传感器平台：`http://iot.lwbsq.com`
- 需要账号密码认证

## API接口说明

### 检测数据API

#### 1. 接收检测数据

**端点**：`POST /receive_detection`

**描述**：接收来自PIVdata2.py的检测数据

**请求体**：
```json
{
  "title": "检测标题",
  "message": "检测消息内容",
  "type": "mill_change",  // 可选，默认"mill_change"
  "url": "http://example.com",  // 可选，任务URL
  "user": "user123"  // 可选，任务用户
}
```

**响应**：
```json
{
  "status": "success",
  "message": "检测数据接收成功",
  "data_id": 1234567890,
  "timestamp": "2026-01-10T10:30:00"
}
```

**错误响应**：
```json
{
  "error": "错误信息"
}
```

**状态码**：
- `200` - 成功
- `400` - 请求参数错误
- `500` - 服务器错误

**调用方**：PIVdata2.py

---

#### 2. 获取检测数据

**端点**：`GET /get_detections`

**描述**：获取检测数据列表（供Android APP使用）

**查询参数**：
- `type`：数据类型（`all`、`unread`），默认`all`
- `limit`：返回数量限制，默认`50`

**示例**：
```
GET /get_detections?type=all&limit=100
```

**响应**：
```json
{
  "status": "success",
  "data": [
    {
      "id": 1234567890,
      "timestamp": "2026-01-10T10:30:00",
      "title": "检测标题",
      "message": "检测消息",
      "type": "mill_change",
      "status": "unread",
      "url": "http://example.com",
      "user": "user123"
    }
  ],
  "total_count": 1,
  "timestamp": "2026-01-10T10:35:00"
}
```

**调用方**：Android APP

---

#### 3. 标记已读

**端点**：`POST /mark_read`

**描述**：标记检测数据为已读

**请求体**：

**响应**：
```json

}
```

**错误响应**：
```json
{
  "status": "error",
  "message": "数据不存在"
}
```

**调用方**：Android APP

## 配置说明

### 配置文件

项目使用统一的配置管理（`shared.config`），支持以下配置方式：

1. **环境变量**
2. **config.json文件**
3. **默认值**

### Flask配置

**配置项**：
- `flask.host`：服务器地址，默认`0.0.0.0`
- `flask.port`：端口号，默认`5000`
- `flask.debug`：调试模式，默认`False`
- `flask.base_url`：基础URL

**示例配置（config.json）**：
```json
{
  "flask": {
    "host": "0.0.0.0",
    "port": 5000,
    "debug": false,
    "base_url": "http://101.34.158.244:5000"
  }
}
```

### 传感器配置

**配置项**：
- `sensor.login_name`：传感器平台登录名
- `sensor.password`：传感器平台密码

**环境变量**：
- `IOT_LOGIN_NAME`：传感器登录名
- `IOT_PASSWORD`：传感器密码

**示例配置（config.json）**：
```json
{
  "sensor": {
    "login_name": "your_login",
    "password": "your_password"
  }
}
```

### 仓储配置

**配置项**：
- `max_history`：最大历史记录数，默认`1000`

**位置**：`interface/api.py`
```python
repository = InMemoryDetectionRepository(max_history=1000)
```

## 部署指南

### 开发环境部署

#### 1. 环境要求
- Python 3.7+
- pip

```bash
cd flask_server
```

```bash
# 方式1：环境变量

{
  "sensor": {
    "login_name": "your_login",

  }
```

```bash
python main.py
```

---

### 生产环境部署

#### 1. 使用Gunicorn

**安装Gunicorn**：
```bash
pip install gunicorn
```

**启动命令**：
```bash
cd flask_server
gunicorn -w 4 -b 0.0.0.0:5000 main:app
```

**参数说明**：
- `-w 4`：4个工作进程
- `-b 0.0.0.0:5000`：绑定地址和端口
- `main:app`：应用入口

#### 2. 使用systemd（Linux）

创建服务文件 `/etc/systemd/system/flask-server.service`：
```ini
[Unit]
Description=Flask Detection Data Server
After=network.target

[Service]
Type=simple
User=your_user
WorkingDirectory=/path/to/flask_server
Environment="PATH=/path/to/venv/bin"
ExecStart=/path/to/venv/bin/gunicorn -w 4 -b 0.0.0.0:5000 main:app
Restart=always

[Install]
WantedBy=multi-user.target
```

**启动服务**：
```bash
sudo systemctl enable flask-server
sudo systemctl start flask-server
sudo systemctl status flask-server
```

#### 3. 使用Nginx反向代理（可选）

**Nginx配置**：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 开发指南

### 添加新的API端点

#### 1. 在Interface层添加路由

编辑 `interface/api.py`：
```python
@app.route('/new_endpoint', methods=['GET'])
def new_endpoint():
    """新端点描述"""
    try:
        # 调用应用层服务
        result = some_service.some_method()
        return jsonify(result)
    except Exception as e:
        return jsonify({'error': str(e)}), 500
```

#### 2. 在Application层添加服务方法

编辑 `application/some_service.py`：
```python
class SomeService:
    def some_method(self) -> Dict[str, Any]:
        # 使用领域层
        entity = create_something(...)
        # 通过仓储保存
        saved = self.repository.add(entity)
        return {...}
```

#### 3. 在Domain层添加领域逻辑

编辑 `domain/rules.py`：
```python
def create_something(...) -> SomeEntity:
    """领域规则：创建实体"""
    # 业务规则
    return SomeEntity(...)
```

### 添加新的检测类型

#### 1. 在Domain层添加枚举值

编辑 `domain/value_objects.py`：
```python
class DetectionType(str, Enum):

```

#### 2. 更新业务规则（如需要）

编辑 `domain/rules.py`，添加新类型的处理逻辑。

### 替换存储实现

当前使用内存存储，可以替换为数据库存储：

#### 1. 创建新的仓储实现

```python
# infrastructure/database_repository.py
class DatabaseDetectionRepository:
    def add(self, detection: Detection) -> Detection:
        # 数据库保存逻辑
        ...
    
    def get_all(self, limit: int) -> List[Detection]:
        # 数据库查询逻辑

```

#### 2. 在Interface层替换

编辑 `interface/api.py`：

## 常见问题

### 1. 服务器启动失败

#### 问题：`ModuleNotFoundError: No module named 'interface'`

**原因**：不在正确的目录下运行，或Python路径问题

**解决方案**：
```bash
# 确保在flask_server目录下运行
cd flask_server
python main.py

# 或使用绝对路径
python /path/to/flask_server/main.py
```

#### 问题：端口被占用

**解决方案**：
```bash
# Windows
netstat -ano | findstr :5000
taskkill /PID <PID> /F

# Linux/Mac
lsof -i :5000
kill <PID>
```

### 2. 数据接收失败

#### 问题：返回500错误但数据已保存

**原因**：JSON序列化失败（已修复）

**解决方案**：
- 检查服务器日志，查看具体错误
- 数据已保存，可以正常查询
- 已实现多层错误恢复机制

#### 问题：`ValueError: 'xxx' is not a valid DetectionType`

**原因**：检测类型不在枚举中

**解决方案**：
- 检查 `domain/value_objects.py` 中的 `DetectionType` 枚举
- 添加缺失的类型值

### 3. 传感器数据获取失败

#### 问题：`传感器账号密码未配置`

**原因**：未配置传感器凭证

**解决方案**：
```bash
# 方式1：环境变量
export IOT_LOGIN_NAME="your_login"
export IOT_PASSWORD="your_password"

# 方式2：config.json
{
  "sensor": {
    "login_name": "your_login",
    "password": "your_password"
  }
}
```

#### 问题：传感器平台连接失败

**原因**：网络问题或账号密码错误

**解决方案**：
- 检查网络连接
- 验证账号密码是否正确
- 查看服务器日志

### 4. Android APP无法获取数据

#### 检查清单

1. **服务器是否运行**
   ```bash
   curl http://your-server:5000/health
   ```

2. **数据是否已保存**
   ```bash
   curl http://your-server:5000/get_detections?type=all&limit=1
   ```

3. **APP配置是否正确**
   - 检查 `AppConfig.java` 中的 `FLASK_BASE_URL`
   - 确认服务器地址和端口正确

4. **网络连接**
   - 确认APP可以访问服务器
   - 检查防火墙设置

### 5. 性能问题

#### 内存占用过大

**原因**：历史数据过多

**解决方案**：
- 调整 `max_history` 参数（默认1000）
- 定期清理旧数据
- 考虑使用数据库存储

## 数据流说明

### 检测数据流程

```
1. PIVdata2.py 检测到变化
   ↓
2. 调用 send_to_flask() 发送数据
   ↓
3. POST /receive_detection
   ↓
4. Interface层：解析HTTP请求
   ↓
5. Application层：DetectionService.receive_detection()
   ↓
6. Domain层：create_detection() 创建实体
   ↓
7. Infrastructure层：Repository.add() 保存数据
   ↓
8. Android APP 请求数据
   ↓
9. GET /get_detections
   ↓
10. Application层：DetectionService.get_detections()
   ↓
11. Infrastructure层：Repository.get_all()
   ↓
12. 返回数据给Android APP
```

### 传感器数据流程

```
1. Android APP 请求传感器数据
   ↓
2. GET /api/realtime
   ↓
3. Application层：SensorService.get_realtime_data()
   ↓
4. Infrastructure层：sensor_client.get_token()
   ↓
5. 外部传感器平台：获取令牌
   ↓
6. Infrastructure层：sensor_client.get_realtime_data()
   ↓
7. 外部传感器平台：返回数据
   ↓
8. Application层：数据格式转换
   ↓
9. 返回数据给Android APP
```
